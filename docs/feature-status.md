# TiebaLite 功能现状清单

最后更新：2026-04-25

这份清单用于说明当前版本中，哪些功能适合日常使用，哪些功能需要登录，哪些功能仍建议谨慎开启。TiebaLite 当前定位是阅读优先的非官方贴吧客户端，不承诺覆盖贴吧官方客户端的全部互动能力。

当前默认策略：

- 主支持范围为 Android 10+
- 优先保证浏览、搜索、吧页、帖子页和本地阅读辅助体验
- 涉及账号写入、自动操作或可能触发风控的功能，默认保持保守
- 未经过测试号完整验证的账号功能，不作为默认推荐入口宣传

| --- | --- | --- | --- |

## 适合日常使用

- 阅读优先的信息架构：
  主导航已收口为 `首页 / 动态 / 我`；消息下沉到用户页次级入口，首页不再直接暴露一键签到，吧页已关注场景下也不再把签到按钮放在高可见位置。
- 基础阅读链路：
  首页、动态、全局搜索、吧页、帖子页是当前版本重点维护的阅读体验。
- 论坛页基础面板：
  当前已形成 `最新 / 精华 / 吧内搜索` 的可扩展结构。
- 话题链路：
  话题榜可以进入，话题详情已经有独立路由与页面。
- 帖子页内容展示：
  图片、视频、外链、楼中楼、IP 属地等内容阅读能力已经纳入日常浏览体验。
- 版本状态提示：
  设置页会说明当前定位；关于页也同步显示同一口径的版本说明。
- 入口提示：
  需要登录或存在风险的入口会显示更明确的状态，不再让用户进入空白页或难以理解的错误页。
- 用户页导航：
  消息入口会直接把用户送去登录或账号管理；收藏和服务中心等完整会话能力不在未登录状态下直接打开。
- WebView 与站内链接分流：
  Tieba 贴子/吧链接会优先回到原生页，`/mo/q/checkurl` 会先解析真实目标，再决定是原生跳转、留在 WebView，还是外跳到系统浏览器 / Custom Tabs。

## 仍在补齐

- 账号功能仍需要更多测试号样本：
  登录结果已经统一收口到 `SessionHealth`，通知页和手动签到在登录信息不完整时会明确拦截并给出提示，退出/切号也会同步清理未读和后台轮询。
- 论坛内容维度仍落后于当前贴吧网页：
  现阶段只有 `最新 / 精华 / 吧内搜索`，还没有补齐更接近网页的 `图片 / 视频 / 推荐` 等内容维度。
- 通知中心完整度仍有限：
  当前重点是 `回复我 / @我 / 赞过我`。其中“赞过我”已经改成 `agree_list / agreeme` 主字段优先，仅在字段缺失时才回退到历史兼容字段，并会显示兼容提示。
- 手动签到已具备保守前置反馈：
  完整登录状态下会正常入队；未登录、Cookie 不完整、账号信息不完整时会立刻提示当前状态。
- Android 平台基线仍停在 SDK 34：
  当前目标是先把现代化兼容层收口，再单独处理 target/compile SDK 升级。
- 自定义交互兼容层仍需继续迁移：
  placeholder、图片渐变、bottom sheet、swipe threshold 已经集中到项目内部封装，底层仍有旧 API 兼容代码，后续要继续向新版 Compose/Foundation 迁移。

## 高风险功能

- 自动签到：
  默认关闭。它会进行自动化请求，建议先使用测试号验证后再长期启用。
- 回帖：
  进入前会强制显示风险提示，并检查登录信息完整性。没有测试号验证前，不建议使用主号尝试。
- 发帖：
  当前版本不优先补齐完整发帖流程。
- 未完成真实账号验证的接口分支：
  默认不作为常规入口开放。

## 暂不承诺

- 完整发帖/回帖体验
- 现行贴吧网页的全量论坛内容维度对齐
- 使用主号完成登录、通知、签到等高风险链路验证
- Android 15/16 上的最终 target SDK 升级结果

## 当前建议验证顺序

1. 先验证阅读链路：
   首页、热榜、搜索、吧页、帖子页、图片、视频、外链。
2. 再用测试号验证账号功能：
   登录、会话健康、通知字段样本、手动签到、退出/切号清理。
3. 最后再决定是否启用高风险功能：
   自动签到、回帖、发帖。

## 已知技术债

- **P1: 构建工具链锁定在 Android 14** — `compileSdk`/`targetSdk` = 34，未来升级到 SDK 35/36 可能引发 edge-to-edge 布局回归，需先审计 `ImmersionBar`/`fitsSystemWindows` 用法。
- **P1: "赞过我"通知解析仍是启发式兼容** — `agree_list`/`agreeme` 作为主字段，缺失时回退到历史字段并显示兼容提示；需要测试号样本确认真实字段后精简。
- **P1: 账号链路缺少真实账号端到端验证** — 登录、通知、签到、退出/切号的守卫逻辑已就位，但仍待测试号闭环确认。
- **P1: 论坛维度落后于网页版** — 当前只有 `最新 / 精华 / 吧内搜索`，缺少 `图片 / 视频 / 推荐` 等内容维度 tab。
- **P2: cleartext traffic 全局开启** — manifest 仍声明 `usesCleartextTraffic="true"`，主要 API 已迁移 HTTPS，待确认是否可收紧。
- **P2: 旧 OEM 兼容 manifest flags** — `largeHeap`、`notch_support`、`EasyGoClient` 等仍在，需逐项验证后清理。
- **P2: 设备身份兼容层仍是 shim** — 已不泄露真实硬件 ID，但 `AID`/`CUID`/`android_id` 等字段仍以假值填充，需配合真实账号验证。
- **P3: 旧 ViewPager/Fragment 基类疑似死代码** — `BaseFragment` 无活跃子类，待确认后删除。
